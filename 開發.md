# DMS 系統架構總覽
> 版本 v2.0 | 2026-07-21 | 給全體工程師（前端/後端/資料庫）共同閱讀的入口文件

## 1. 系統目的
文件管理系統(DMS)串接 SAP/MES/WMS/HMRB/QMS 等來源系統，自動整理資料/檔案、依範本生成標準文件(PDF+Word)、
同步至 SharePoint，並透過 MCP 協議提供 AI Agent 查詢/下載，同時提供內部人員操作介面。

## 2. 模組總覽與對應開發文件

| 模組 | 說明 | 對應文件 | 負責工程師類型 |
|---|---|---|---|
| 前端介面 | 資料夾樹、文件列表、範本設定、功能指南頁面 | 02_前端開發規格.md | 前端工程師 |
| 後端服務 | 業務邏輯、REST API、文件生成引擎 | 01_後端開發規格.md | 後端工程師(Python) |
| MCP Server | AI Agent 呼叫介面 | 04_MCP規格.md | 後端工程師(Python) |
| 資料庫 | SQLite(系統資訊) + MSSQL(中繼DB讀取) | 03_資料庫規格.md | 後端/DBA |

## 3. 模組邊界原則（模組化但非完全隔離）
- 前端**只透過 REST API** 與後端溝通，不直接存取資料庫，前端工程師只需依 02 文件的 API contract 開發，不需了解後端內部邏輯。
- 後端 REST API 的 request/response 格式為前後端之間的「合約」，任何欄位變更需同步更新 02、01 兩份文件。
- MCP Server 與 REST API 共用同一套後端業務邏輯(service層)，但對外是兩個獨立端點（`/api/mcp` vs `/api/v1/...`），互不影響。
- 資料庫變更（新增欄位/表）由後端工程師提出，統一更新 03 文件後，前端才依新增欄位調整畫面。

## 4. 主機與部署資訊
- Host: 192.168.31.211（Linux），SSH port 22 僅供部署管理。
- 應用服務對外 port：待定（建議如 8000），需與IT確認防火牆。
- 技術棧：後端 Python + FastAPI；前端 React；DB：SQLite + MSSQL(既有中繼DB，唯讀)。

## 01_後端開發規格.md

```markdown
# 01_後端開發規格
> 給後端工程師使用（Python） | v1.0
> 依賴：03_資料庫規格.md（表結構）、04_MCP規格.md（MCP端點細節）

## 1. 技術棧
- FastAPI + Uvicorn
- SQLAlchemy（同時操作 SQLite 與 MSSQL唯讀連線）
- python-docx（Word生成）、WeasyPrint 或 ReportLab（PDF生成）
- APScheduler（排程：來源系統資料拉取、心跳）
- python-docx / docxtpl 建議用於範本佔位變數替換

## 2. 模組劃分（依職責拆分，供不同工程師平行開發）

| 子模組 | 說明 | 主要依賴表 |
|---|---|---|
| source_integration | 對接 SAP/MES/WMS/HMRB/QMS，取資料/檔案並分類落地 | documents, categories |
| template_engine | 範本設定CRUD、欄位對照與轉換規則執行 | templates, field_mappings, template_versions |
| doc_generator | 讀取中繼DB資料+範本，套用轉換規則，產出PDF/Word | templates, field_mappings, documents |
| sharepoint_sync | 呼叫PA接口，將產出文件同步至SharePoint，更新連結 | documents |
| mcp_server | 對外 JSON-RPC 端點，供AI Agent呼叫（詳見04文件） | documents, templates |
| rest_api | 對外 REST端點，供前端呼叫 | 全表 |
| api_logger | 中介層(middleware)，攔截所有請求寫入 api_logs | api_logs |
| dev_spec_generator | 動態彙整功能指南內容 | templates, mcp manifest |

## 3. REST API 規格（前端 <-> 後端合約）

> Base path: `/api/v1`
> 認證：Header `Authorization: Bearer <token>`，token內含角色(role)資訊

### 3.1 分類 Categories

`GET /api/v1/categories` — 取得分類樹（含sort_order）
```json
[
  { "category_id": "c1", "category_name": "配方審核", "stage": "中", "parent_id": null, "sort_order": 1,
    "children": [] }
]
```

`PATCH /api/v1/categories/{category_id}/order` — 更新排序（拖曳後呼叫）

```json
{ "sort_order": 2, "parent_id": "c0" }
```


### 3.2 文件 Documents

`GET /api/v1/documents?category_id=&keyword=&date_from=&date_to=&page=`

```json
{ "total": 42, "items": [
  { "doc_id": "d001", "doc_name": "原料COA_2026070101", "category_id": "c5",
    "format": "PDF", "sharepoint_link": "https://...", "status": "已同步",
    "created_at": "2026-07-20T10:00:00" }
]}
```

`GET /api/v1/documents/{doc_id}/download` — 回傳檔案本體(供人員介面單筆下載)

`POST /api/v1/documents/batch-download` — 批次下載（Base64）

```json
{ "doc_ids": ["d001", "d002"] }
```

Response:

```json
{ "items": [
  { "doc_id": "d001", "filename": "xxx.pdf", "content_base64": "..." }
]}
```

限制：單次總大小 <= 20MB 或 <=10份，超過回傳 409 並附SharePoint資料夾連結

`DELETE /api/v1/documents/{doc_id}` — 僅 Admin 角色可呼叫，否則回 403

### 3.3 範本 Templates（獨立文件生成設定）

`GET /api/v1/templates?category_id=`

`POST /api/v1/templates` — 建立範本 (multipart/form-data)

```json
{
  "template_name": "原料COA範本",
  "category_id": "c5",
  "output_format": "BOTH",
  "template_file": "<上傳檔案>"
}
```

`PUT /api/v1/templates/{template_id}` — 覆蓋更新（自動寫入 template_versions 保留舊版本）

`GET /api/v1/templates/{template_id}/versions` — 查詢版本歷史

`POST /api/v1/templates/{template_id}/field-mappings` — 設定欄位對照與轉換規則

```json
{
  "placeholder_name": "{{批號}}",
  "source_field": "batch_no",
  "transform_rule": { "type": "date_format", "input_format": "YYYYMMDD", "output_format": "YYYY-MM-DD" },
  "sort_order": 1
}
```

`GET /api/v1/templates/{template_id}/preview` — 帶入範例資料預覽生成結果

### 3.4 功能指南（動態產生，不可人工編輯）

`GET /api/v1/dev-spec` — 彙整回傳：目前MCP工具清單(含schema)、啟用中範本對應分類代碼、REST API端點清單

```json
{
  "mcp_tools": [ { "name": "get_document", "params_schema": {}, "response_schema": {} } ],
  "active_categories": [ { "category_id": "c5", "category_name": "配方審核" } ],
  "rest_endpoints": [ { "path": "/api/v1/documents", "method": "GET" } ]
}
```


### 3.5 使用者/角色

`GET /api/v1/me` — 取得目前登入者角色，供前端決定UI顯示

## 4. 文件生成引擎邏輯（doc_generator）

```
1. 接收 trigger（來源系統新資料到位 / 手動重新生成）
2. 依 category_id 查詢 templates，取得啟用中的範本
3. 依 field_mappings 逐筆讀取中繼DB來源欄位
4. 依 transform_rule 執行轉換（日期格式/合併/條件式），支援多規則依sort_order鏈式執行
5. 若有 source_doc_rules，額外抓取來源系統原始檔案一併附上
6. 套用範本檔案(docxtpl)產出 Word；轉換或另用ReportLab產出PDF
7. 寫入 documents 表(status=已生成)
8. 呼叫 sharepoint_sync 模組同步 -> 更新 sharepoint_link, status=已同步
```


## 5. 錯誤處理原則

- 所有REST端點錯誤回應統一格式：`{ "error_code": "xxx", "message": "..." }`
- 中繼DB連線失敗需重試(最多3次)並記錄log，不可讓文件生成排程整組crash。
- 範本轉換規則執行失敗，該筆記錄標記 status=失敗，不中斷其他筆處理。


## 6. 權限中介層

- `Admin`：全功能，包含刪除/下架文件、範本管理。
- `Create`：分類建立、文件生成觸發、範本設定；不可刪除/下架文件。
- `View`：僅 GET 類端點，其餘回403。

```

1. 基本資訊：範本名稱、所屬分類、輸出格式

3. 欄位對照設定：
   - 左側顯示範本內偵測到的佔位變數
   - 右側每個變數設定來源欄位 + 轉換規則（日期格式/多欄位合併/條件式取值）

4. 來源文件規則設定（選填）：文件類型、命名規則(regex)
5. 預覽：GET /templates/{id}/preview

### 版本歷史
- 上傳新檔提示會建立新版本，GET /templates/{id}/versions 查看歷史

## 5. 頁面：DMS功能指南 /dev-guide
- 純渲染頁，呼叫 GET /dev-spec，依回應顯示：MCP工具清單、啟用分類代碼、REST API端點清單
- 重要：此頁不應有任何硬編碼說明文字，全部內容來自API回應

## 6. 共用元件
- `<StatusTag>`：狀態標籤，依status顯示不同顏色
- `<FormatTag>`：PDF/WORD圖示標籤
- `<RoleGate roles={['Admin']}>`：權限包裝元件

## 7. 前後端串接注意事項

- 錯誤格式統一 { error_code, message }，前端全域攔截顯示toast
- 批次下載超限(409)需處理並提示改用SharePoint連結

# 04_MCP規格
> 給後端工程師使用（MCP Server部分） | v1.0

## 1. 定位
DMS對AI Agent開放的介面，走JSON-RPC，不走一般REST，與rest_api模組共用同一套service層邏輯。

## 2. 端點
- POST /api/mcp — 統一入口，處理三種method：initialize、tools/list、tools/call

## 3. 註冊流程（開機時執行）
1. 讀取環境變數 DMS_MCP_SHARED_SECRET，未設定則跳過註冊
2. 呼叫 POST http://<AI平台host>/api/ai/mcp-servers/<serverKey>/register，帶manifest
3. 失敗採指數退避重試：5s,15s,30s,60s,120s（之後固定120s）
4. 成功後啟動心跳排程，每5分鐘一次 .../heartbeat

## 4. 工具Manifest定義

### 4.1 get_document
```json
{
  "name": "get_document",
  "description": "依分類或關鍵字查詢文件，回傳SharePoint連結與metadata",
  "input_schema": {
    "type": "object",
    "properties": {
      "category": { "type": "string" },
      "keyword": { "type": "string" }
    }
  },
  "output_schema": {
    "type": "object",
    "properties": {
      "doc_id": {"type": "string"},
      "doc_name": {"type": "string"},
      "sharepoint_link": {"type": "string"},
      "format": {"type": "string"},
      "status": {"type": "string"}
    }
  }
}
```


### 4.2 list_documents

```json
{
  "name": "list_documents",
  "description": "列出符合分類/時間條件的文件清單",
  "input_schema": {
    "type": "object",
    "properties": {
      "category": {"type": "string"},
      "date_from": {"type": "string", "format": "date"},
      "date_to": {"type": "string", "format": "date"}
    }
  }
}
```


### 4.3 batch_download_documents

```json
{
  "name": "batch_download_documents",
  "description": "批次下載，Base64回傳，單次限制:總大小<=20MB或<=10份，超過改回傳資料夾連結",
  "input_schema": {
    "type": "object",
    "properties": {
      "doc_ids": {"type": "array", "items": {"type": "string"}}
    }
  },
  "output_schema": {
    "type": "object",
    "properties": {
      "items": {"type": "array", "items": {
        "type": "object",
        "properties": {
          "doc_id": {"type": "string"},
          "filename": {"type": "string"},
          "content_base64": {"type": "string"}
        }
      }},
      "exceeded_limit": {"type": "boolean"},
      "fallback_folder_link": {"type": "string"}
    }
  }
}
```


## 5. 呼叫路由邏輯

```
tools/call收到請求
  get_document/list_documents -> documents service唯讀查詢 -> 回傳連結
  batch_download_documents -> 檢查大小限制
    在限制內: 讀檔轉Base64回傳
    超過限制: 回傳fallback_folder_link
```


## 6. 與dev-spec的關聯

- 所有manifest定義應以程式內單一來源維護，tools/list回應與GET /api/v1/dev-spec的mcp_tools欄位皆從同一份定義讀取
